# Accessibility (WCAG AA)

## Semantics (ALL interactive widgets)
Every button, icon button, gesture detector, text field, checkbox, radio, switch:
  Semantics(
    identifier: 'feature_action_element',  // for E2E tests
    label: 'Human-readable label',         // for screen readers
    hint: 'What will happen',              // describes the action result
    button: true,                          // if it acts as a button
  )

## Color Contrast
- Normal text (< 18pt): >= 4.5:1 contrast ratio against background.
- Large text (>= 18pt / bold >= 14pt): >= 3:1.
- UI components (borders, icons): >= 3:1.
- Never rely on color alone to convey meaning — pair with text or icon.

## Touch Targets
- Minimum tap target: 48x48 logical pixels.
- If visual size is smaller, wrap in SizedBox(width: 48, height: 48).

## Focus Management
- ExcludeSemantics() for purely decorative elements.
- FocusNode management for form fields — FocusScope.of(context).nextFocus().
- Modal dialogs: trap focus within. Return focus on close.

## Screen Reader Support
- Images: Image(semanticsLabel: '...') or ExcludeSemantics() for decorative.
- Form errors: use ErrorText widget with Semantics(liveRegion: true).
- Dynamic content updates: use SemanticsService.announce().

## Testing
- Widget tests: verify Semantics tree with find.bySemanticsLabel().
- Run /a11y-check before every PR merge.
